# 文档索引

本仓库只维护两类文档：

- **架构设计文档**：描述系统边界、组件关系、数据流和扩展点
- **使用文档**：描述安装、配置、命令、测试与运维方式

不再维护以下类型的文档：

- 按源码文件拆分的实现说明
- 单次重构总结、阶段性完成报告
- 一次性问题修复纪要
- 与当前代码脱节的未来实施计划
- 独立的功能设计草案文档入口

代码级细节以源码、测试和 CLI 帮助输出为准。

## 当前文档集合

除以下三份核心文档外，`docs/` 下其余专题文档仅作为**当前实现的补充说明或归档说明**存在，不应视为独立维护主线。

- `docs/QUICK_START.md`
  - 面向使用者
  - 包含安装、初始化、配置、常用命令和能力边界

- `docs/ARCHITECTURE.md`
  - 面向维护者
  - 描述运行时结构、Agent 循环、provider、工具系统、持久化与扩展点

- `docs/DEVELOPMENT.md`
  - 面向贡献者
  - 包含构建、测试、E2E、日志与提交流程

另有一份新手导读文档：

- `docs/OPEN_SOURCE_LEARNING_PATH.md`
  - 面向初学贡献者
  - 给出本仓库的阅读路径与上手任务

新增补充参考文档（按当前源码行为维护）：

- `docs/CONFIGURATION.md`
  - 配置字段、默认值、校验与行为边界
- `docs/CLI_REFERENCE.md`
  - CLI 子命令、参数、聊天内建命令
- `docs/DEPLOYMENT.md`
  - gateway 运行方式、目录、通道配置、GitHub Release 发版与运维排查

## 专题文档说明

当前保留的专题文档，例如 provider、streaming、memory 等说明文档，仅用于补充解释某一块**已经落地的当前实现边界**，或明确标注“当前未实现 / 已归档”的状态。

维护这些专题文档时应遵循：

- 内容必须和当前代码一致
- 只补充核心文档不便展开的局部实现说明
- 不再承载阶段计划、重构总结或未来方案设计
- 如果核心文档已经足够表达，应继续收敛而不是扩写

## 更新规则

- 当 **CLI、配置、provider、工具、渠道能力** 变化时，更新 `docs/QUICK_START.md`
- 当 **组件边界、数据流、扩展方式** 变化时，更新 `docs/ARCHITECTURE.md`
- 当 **开发流程、测试命令、调试方法** 变化时，更新 `docs/DEVELOPMENT.md`

## 维护原则

- 优先维护“当前真实行为”，不记录历史过程
- 优先描述“系统边界”和“如何使用”，不追踪逐文件实现细节
- 新增能力时，优先更新现有文档，不再新增同类“总结型”或“设计草案型”文档
